# DID 기반 응급 의료 신원 인증 시스템

> 48시간 Open DID 해커톤 데모. **SD-JWT VC + 한국 mDL Reference Issuer + 정적 DDI 룩업**으로 30초 안에 "선택적 공개"의 인간 가치를 시연.

## 데모 실행

```bash
npm install
npm run dev        # → http://localhost:5173/wallet
npm test           # 6개 vitest 케이스
npm run build      # 프로덕션 빌드 (Vercel/Netlify 정적 배포 가능)
```

두 탭으로 시연:
- `/wallet` — 환자 모바일 지갑 (mDL · 혈액형 · 알레르기 VC 3개, 응급 모드 토글 → QR + 클립보드 자동 복사)
- `/doctor` — 의사 화면 (QR 스캔 또는 페이스트 → SD-JWT VP 검증 → 마스킹된 클레임 + DUR 충돌 경고)

## 30초 데모 시나리오

1. **0-10초** — wallet 진입, mDL "🇰🇷 Korean mDL Reference Issuer (Demo)" 배지 노출 (한국 특화 angle).
2. **10-20초** — 응급 모드 ON → QR 생성 → doctor에서 스캔/페이스트 → 혈액형 A+, 페니실린 알레르기만 표시. 이름·주민번호·주소는 ●●● 마스킹.
3. **20-28초** — 의사 "Amoxicillin" 선택 → 빨간 충돌 경고 (페니실린 알레르기). 미등록 약물(Vancomycin 등)은 회색 "수동 확인" 배지.
4. **28-30초** — "환자가 의식 없으면?" 슬라이드 1장 (역방향 호출 아키텍처는 슬라이드만, 본 코드 범위 밖).

## 기술 스택

- Vite + React + TypeScript + React Router + Tailwind
- SD-JWT-like wrapper 자체 구현 (`src/lib/sdjwt.ts`) on `@noble/curves/ed25519` + `@noble/hashes`. 프로덕션은 `@sd-jwt/sd-jwt-vc` 권장.
- QR: `qrcode` (생성) + `html5-qrcode` (스캔, paste fallback 항상 노출)
- DUR: 한국 DUR 참고 정적 룩업 (5종 알레르기 × 10종 응급 약물)

## 보안 주의

본 코드의 모든 issuer/holder 키는 `src/data/issuer-keys.ts`에 hardcode된 데모 키입니다. 프로덕션 사용 금지. 한국 mDL/행안부 키 인프라와 무관한 reference implementation입니다.

## 아이디어 개요

모바일 신분증과 DID(Decentralized Identity), 블록체인 기술을 활용하여 응급 상황에서 필요한 의료 정보를 빠르게 확인하고, 동시에 마약성 의약품 오남용 및 불법 처방을 방지하는 의료 인증 플랫폼.

사용자는 자신의 의료 정보를 DID 지갑에 직접 보관하며, 병원은 환자의 동의 혹은 응급 권한을 통해 필요한 정보만 열람할 수 있다.

# 핵심 기능 1 : DID 기반 응급실 신원 인증

## 문제점

응급 상황에서는 환자의 정보를 빠르게 확인하기 어렵다.

예시:

* 보호자 연락 지연
* 알레르기 여부 미확인
* 혈액형 확인 지연
* 복용 중인 약 정보 부족
* 기저질환 정보 부족

이로 인해 응급 처치가 늦어지거나 위험한 약물이 투여될 가능성이 존재한다.

---

## 해결 방식

사용자는 모바일 DID 지갑에 자신의 의료 정보를 저장한다.

병원은 환자의 QR 혹은 DID 인증을 통해 필요한 정보만 선택적으로 열람할 수 있다.

예시:

* 페니실린 알레르기
* 당뇨 환자 여부
* 혈액형
* 응급 연락처
* 현재 복용 약
* 기저질환 정보

개인정보 전체를 공개하지 않고 필요한 정보만 공개할 수 있다는 점에서 DID 철학과 매우 잘 맞는다.

---

## AI 활용

AI는 환자의 의료 데이터를 기반으로 위험 요소를 분석한다.

예시:

* 복용약 충돌 탐지
* 응급 위험도 분석
* 특정 약물 알레르기 경고
* 환자 상태 기반 응급 우선순위 추천

---

## 블록체인 활용

* 의료 정보 열람 기록 저장
* 위변조 방지
* 병원의 접근 기록 투명성 확보
* 환자 본인의 데이터 통제 강화

---

# 핵심 기능 2 : 마약성 의약품 처방 인증 및 오남용 방지 시스템

## 문제점

현재 일부 병원에서는 다음과 같은 문제가 존재한다.

* 타인의 신분을 도용한 마약성 진통제 처방
* 여러 병원을 돌며 중복 처방받는 행위
* 허위 신원 정보 사용
* 의료진이 환자의 과거 처방 이력을 빠르게 파악하기 어려움

이로 인해 마약성 의약품 오남용 및 불법 유통 문제가 발생할 수 있다.

---

## 해결 방식

환자의 DID 의료 신원에는 다음과 같은 정보가 포함된다.

예시:

* 마약성 의약품 처방 이력
* 최근 처방 병원 기록
* 중복 처방 여부
* 특정 약물 위험군 여부
* 의료진 인증 정보

병원은 환자의 DID 인증 후 필요한 정보만 조회할 수 있다.

이를 통해:

* 타인 명의 도용 방지
* 중복 처방 탐지
* 불법 처방 감소
* 의료진의 빠른 판단 지원

이 가능해진다.

---

## AI 활용

AI는 환자의 처방 패턴을 분석하여 이상 행동을 탐지한다.

예시:

* 짧은 기간 내 다수 병원 방문
* 비정상적 약물 처방 패턴
* 특정 약물 반복 요청
* 위험 환자군 자동 탐지

AI는 의료진에게 위험 경고를 제공하여 오남용 가능성을 줄인다.

---

## 블록체인 활용

* 마약성 의약품 처방 이력 위변조 방지
* 병원 간 신뢰 가능한 처방 검증
* 환자 인증 기록 저장
* 의료기관 접근 기록 관리

# 왜 블록체인이 반드시 필요한가?

## 핵심 이유

의료 신원 정보와 마약성 의약품 처방 기록은 절대 위조되거나 임의 수정되어서는 안 된다.

또한 특정 병원이나 기관 하나만 완전히 신뢰하는 중앙화 구조에는 한계가 존재한다.

따라서 여러 기관이 함께 신뢰할 수 있는 구조와 위변조가 어려운 데이터 관리 방식이 필요하며, 이를 해결하기 위해 블록체인 기술이 사용된다.

---

# 1. 중앙화된 의료 시스템의 한계

현재 의료 시스템은:

* 병원 서버
* 약국 서버
* 정부 시스템

등이 각각 분리되어 운영되는 경우가 많다.

이러한 중앙화 구조에서는 다음과 같은 문제가 발생할 수 있다.

## 문제점

* 병원 간 데이터 연동이 어려움
* 특정 기관 서버 해킹 시 위험 발생
* 데이터 조작 가능성 존재
* 기록 삭제 및 수정 가능
* 누가 언제 열람했는지 불투명
* 기관마다 데이터 신뢰도가 다름

예시:

* 특정 병원이 처방 기록을 수정하는 경우
* 의료 기록 일부가 누락되는 경우
* 접근 로그가 삭제되는 경우

이러한 문제는 의료 정보의 신뢰성을 떨어뜨릴 수 있다.

---

# 2. 블록체인을 사용하면 해결되는 점

블록체인을 사용하면 모든 의료 인증 및 처방 기록이 변경이 어려운 형태로 저장된다.

## 블록체인의 특징

* 데이터 위변조 방지
* 수정 이력 기록 유지
* 여러 기관이 동일 데이터 공유
* 접근 기록 추적 가능
* 특정 기관 단독 조작 어려움

즉:

> 의료 기록의 신뢰성을 기술적으로 보장할 수 있다.

---

# 3. 마약성 의약품 문제에서 블록체인이 중요한 이유

현재 마약성 의약품 분야에서는 다음과 같은 문제가 존재한다.

## 현재 문제

* 여러 병원을 돌며 중복 처방받는 행위
* 타인 신분 도용
* 일부 병원의 기록 누락
* 병원 간 정보 공유 지연
* 처방 이력 확인의 어려움

---

## 블록체인 기반 시스템에서는

예시:

병원 A에서:

* 마약성 진통제 처방

→ 해당 기록이 블록체인에 저장됨

병원 B에서는:

* 환자의 DID 인증 후 처방 기록 검증 가능

이를 통해:

* 최근 중복 처방 여부 확인
* 위험 환자 패턴 탐지
* 신분 도용 탐지
* 불법 처방 가능성 감소

가 가능해진다.

즉:

> 병원 간 신뢰 가능한 처방 공유 체계를 구축할 수 있다.

---

# 4. 일반 DB와 블록체인의 차이점

## 일반 DB 방식

* 병원별 개별 관리
* 중앙 서버 신뢰 필요
* 관리자 수정 가능
* 로그 삭제 가능
* 특정 기관 장애 시 문제 발생

---

## 블록체인 방식

* 여러 기관 공동 검증
* 수정 이력 유지
* 위변조 어려움
* 투명한 접근 기록 관리
* 탈중앙 기반 신뢰 구조

즉:

> 신뢰를 특정 기관이 아니라 기술 구조 자체로 보장한다.

---

# 5. DID와 블록체인의 관계

DID의 핵심은:

> 사용자가 자신의 신원 정보를 직접 소유하고 통제하는 것

이다.

하지만 DID가 실제로 신뢰를 가지기 위해서는:

* 진짜 병원이 발급했는지
* 위조되지 않았는지
* 유효한 인증인지

를 검증할 수 있어야 한다.

이 역할을 블록체인이 수행한다.

즉:

* DID = 디지털 신원
* 블록체인 = 그 신원이 진짜임을 증명하는 신뢰 인프라

의 관계로 볼 수 있다.

---

# 최종 핵심 정리

본 서비스에서 블록체인은 단순 저장 기술이 아니라:

* 의료 정보 신뢰성 확보
* 마약성 의약품 오남용 방지
* 병원 간 신뢰 가능한 데이터 공유
* 접근 기록 투명성 확보
* DID 기반 신원 검증

을 가능하게 하는 핵심 기술이다.

특히 여러 병원과 약국이 동일한 처방 신뢰 체계를 공유해야 하기 때문에 중앙 서버 기반 방식보다 블록체인 기반 탈중앙 구조가 더 적합하다.
