## Affected Pages/Routes
- `/wallet` — 환자 PWA wallet (모바일 뷰포트 에뮬레이션). 응급 모드 토글, QR 생성, iframe embed 모드, LockScreen 자동 응답
- `/wallet?embed=1` — wallet iframe 모드 (UI 축소). MessageListener active
- `/doctor` — 의사 화면. QR 스캐너 또는 paste, presentation 검증, 마스킹 클레임 표시, 약물 선택, DUR 충돌 경고, "응급 호출" 트리거

## Key Interactions to Verify

### QR 흐름 (메인 데모, 0-25초)
- 환자 wallet 첫 진입 시 mDL + 혈액형 + 알레르기 VC 카드 3개 표시
- 응급 모드 OFF 상태에서 QR 생성 버튼 disabled
- 응급 모드 ON 토글 → SD-JWT VP 생성 → QR canvas 표시 + clipboard 자동 복사
- 의사 페이지에서 QR 스캔 (or paste) → 혈액형 A+, 페니실린 알레르기만 표시
- mDL/이름/주민번호/주소 클레임은 ●●● 마스킹으로 표시 (전체 마스킹)
- 의사가 약물 "Amoxicillin" 선택 → 빨간 경고 "페니실린계 알레르기 환자 — 충돌"

### 역방향 호출 (온세컨드, 25-30초)
- 의사 페이지가 wallet을 `<iframe src="/wallet?embed=1">`로 임베드 (우측 패널)
- "응급 호출" 버튼 클릭 → iframe.contentWindow.postMessage (verifier role VC + nonce 동봉, targetOrigin = origin)
- wallet iframe 내 MessageListener가 event.origin 검증 → verifier role VC 검증 → policy 매칭 → 자동 SD-JWT VP 생성 → window.parent.postMessage
- wallet 화면이 LockScreen 컴포넌트로 전환 + "응급 정책 매칭됨, 자동 응답" 토스트
- 의사 화면에 같은 정보 다시 표시

## Edge Cases

### postMessage 흐름
- 잘못된 origin에서 postMessage 수신 → wallet은 무시, console.warn
- wallet이 3초 안에 응답 안 함 → doctor 화면에 "Wallet이 응답하지 않음 — QR 스캔으로 진행" 토스트 + UI roll back
- policy 매칭 실패 (verifier role이 policy.allow에 없음) → wallet은 응답 안 함, doctor는 timeout 경로

### DUR 룩업
- 알레르기 매칭 약물 선택 (Amoxicillin / Sulfamethoxazole 등) → 빨간 경고
- 룩업 미등록 약물 (Vancomycin 등) → 회색 배지 "⚠️ 데모 룩업에 등록 안 된 약물 — 수동 확인 필요"
- 충돌 없는 매칭 약물 (페니실린 안 든 약) → "✅ 충돌 없음"

### VC 검증
- aud 불일치 (다른 demo session ID) → 빨간 박스 "❌ VC 검증 실패: aud mismatch"
- 만료 시간 지남 (`exp` 과거) → 빨간 박스 "❌ VC 검증 실패: expired"
- 서명 무효 (issuer key 손상 등) → 빨간 박스 "❌ VC 검증 실패: invalid signature"
- trust anchor 미일치 → 빨간 박스 "❌ VC 검증 실패: untrusted issuer"

### QR 인식
- QR 인식 성공 → 자동 처리
- QR 인식 실패 (각도/조명/사이즈) → paste textarea로 fallback. 같은 검증 흐름

### 상태/seed
- localStorage 비어있음 (incognito 첫 진입) → 빌드 타임 patient-vcs.ts seed 자동, 정상 진행
- localStorage 클레임 손상 → 시드 재실행, 정상 진행 (사용자 알림 없음)

## Critical Paths

### Path 1: 30초 메인 데모 (반드시 완주)
1. wallet 진입 → 3 VC 카드 표시
2. 응급 모드 ON → QR 표시
3. doctor 진입 → QR 스캔 (or paste) → 혈액형 + 알레르기 마스킹된 표시
4. 약물 Amoxicillin 선택 → 빨간 충돌 경고
5. 의사 "응급 호출" → wallet iframe LockScreen 토스트 → doctor 같은 정보 다시 표시

### Path 2: 심사 dev tools 검증 (응급 질문 대비)
- doctor 페이지 dev tools console → SD-JWT VP 페이로드 확인
- `~`로 disclosure 구분 시각화 가능
- decoded JWT의 `_sd_alg: sha-256`, `alg: EdDSA` 명시

### Path 3: postMessage 실패 시 graceful fallback
- 의사가 응급 호출 클릭 → wallet 응답 없음 → 3초 후 자동 QR fallback Toast
- 데모 흐름 안 죽음

## Test Implementation 우선순위 (48시간 컨텍스트)

- **6시간 체크포인트 (CRITICAL, smoke test로 검증)**: `sdjwt.ts` round trip (sign → present → verify) + 1개 disclosure
- **24시간 (필수)**: `EmergencyCall.tsx` 3초 timeout 테스트 (fake timers) + `MessageListener.ts` origin 검증 테스트
- **36시간 (좋으면)**: 4개 VC 검증 실패 케이스 (aud/exp/sig/anchor) → 모두 throw 확인
- **시연 직전 (시간 남으면)**: Playwright E2E 1개 — wallet → doctor → 마스킹 표시 + 빨간 경고

해커톤 컨텍스트에서 컴포넌트 단위 테스트·QR wrapper 단위 테스트·scripts 테스트는 데모 결정성에 직접 영향 없으므로 cut.

## Test Frameworks

- **Unit/integration**: `vitest` (Vite 프로젝트 표준)
- **E2E (옵션)**: `playwright` — 시간 남으면 1개 시나리오
- **Fake timers for timeout 테스트**: `vi.useFakeTimers()`
